### **1. Архитектура GAN-моделей**

Генеративно-состязательные сети (Generative Adversarial Networks) были предложены Яном Гудфеллоу (Ian Goodfellow) в 2014 году и представляют собой систему из двух нейронных сетей, обучающихся совместно. В архитектуре GAN выделяют два ключевых компонента: генератор и дискриминатор.

#### **1.1 Генератор (Generator)**
Генератор принимает на вход случайный шум (обычно вектор из случайных чисел) и преобразует его в объект, например изображение. Задача генератора — создать данные, максимально похожие на реальные.

#### **1.2 Дискриминатор (Discriminator)**
Дискриминатор получает на вход либо реальные данные, либо сгенерированные генератором, и должен определить, являются ли они настоящими.

#### **1.3 Принцип обучения**
Обучение GAN можно представить как игру с нулевой суммой: генератор пытается «обмануть» дискриминатор, в то время как дискриминатор пытается отличить подделку от реальных данных. Этот процесс часто описывают с помощью метафоры «фальшивомонетчика и детектива». Генератор выступает в роли фальшивомонетчика, стремящегося создать подделку, а дискриминатор — в роли эксперта, который должен распознать подделку.

В процессе обучения требуется делать два шага оптимизации поочередно: один для дискриминатора, а другой для генератора. В результате обе сети постепенно улучшаются, и после обучения генератор способен создавать реалистичные объекты, которые трудно отличить от настоящих.

Для повышения стабильности обучения были разработаны различные модификации GAN. Наиболее известные из них включают:
* **DCGAN (Deep Convolutional GAN)**: архитектура, использующая сверточные слои для повышения качества генерации изображений.
* **Wasserstein GAN (WGAN)**: использует метрику Вассерштейна для решения проблемы нестабильности обучения исходной GAN.
* **Conditional GAN (cGAN)**: позволяет управлять процессом генерации, добавляя метку класса или другую информацию на вход сетей.
* **CycleGAN**: архитектура, позволяющая переносить стили между доменами без использования парных данных.
* **Progressive GAN**: метод, который начинает обучение с малых разрешений и постепенно их увеличивает.
* **StyleGAN**: архитектура, обеспечивающая непревзойденное качество генерации фотореалистичных изображений с возможностью независимого контроля признаков на разных масштабах.
* **StackGAN**: архитектура, способная генерировать изображения высокого разрешения по текстовому описанию.

Также важным направлением является изменение функции потерь для улучшения обучения GAN. Например, существуют подходы, рассматривающие проектирование функции потерь как задачу оптимизации, решаемую с помощью методов генетического программирования.

### **2. Обзор существующих GAN-систем**

В рамках работы были рассмотрены и протестированы несколько современных систем, основанных на GAN и смежных генеративных подходах.

#### **2.1 Генерация изображений**

**Stable Diffusion**
Это одна из самых популярных систем генерации изображений по текстовому описанию. Среди её возможностей — генерация изображений по тексту, изменение изображений (inpainting) и стилизация. К плюсам можно отнести высокое качество изображений, гибкость и возможность локального запуска, а к минусам — высокие требования к оборудованию и возможные артефакты на создаваемых изображениях.

**DALL-E 3 (OpenAI)**
Одна из самых мощных моделей для генерации изображений по текстовым запросам, отличающаяся высокой детализацией и точностью понимания сложных описаний. Интегрирована с ChatGPT, что упрощает процесс уточнения промптов.

**Midjourney**
Известна своим уникальным художественным стилем и качеством проработки деталей, работающая через Discord.

**Adobe Firefly**
Модель, ориентированная на профессиональных дизайнеров, глубоко интегрированная в экосистему продуктов Adobe (Photoshop, Illustrator).

#### **2.2 Генерация звука и музыки**

**Riffusion**
Система для генерации звука, которая работает через генерацию спектрограмм, затем преобразуемых в аудиосигнал. Среди плюсов — быстрое создание музыки и интересный подход. К недостаткам можно отнести ограниченную длину генерируемых треков и качество, уступающее профессиональному уровню.

**Jukebox (OpenAI)**
Нейросеть, способная генерировать музыку с вокалом в различных стилях и жанрах. Отличительной особенностью является генерация сырого аудиосигнала.

**HiFi-GAN / MelGAN (нейронные вокодеры)**
Специализированные GAN-архитектуры для преобразования спектрограмм в аудиосигнал высокого качества. Используются в системах синтеза речи для придания голосу естественности.

#### **2.3 Генерация видео**

**Runway ML**
Платформа для генерации и редактирования видео с помощью нейросетей. Предоставляет широкий набор инструментов от превращения текста в видео до ретуширования отдельных кадров.

**Sora (OpenAI)**
Модель для генерации видео по текстовому описанию, способная создавать сложные сцены с несколькими персонажами и точной физикой объектов.

**Pika Labs**
Платформа для генерации и редактирования видео, известная своими возможностями по анимации статичных изображений.

### **3. Практическая реализация GAN для генерации рукописных цифр**

Для демонстрации принципов работы GAN была разработана и обучена модель для генерации изображений рукописных цифр из датасета MNIST. Реализация выполнена на Python с использованием библиотеки PyTorch.

#### **3.1 Архитектура модели**

**Генератор (Generator)**
Архитектура генератора представляет собой последовательность полносвязных слоев (для простого GAN) или транспонированных сверточных слоев (для DCGAN). На вход генератор получает вектор случайного шума размерности `latent_dim` (128). Этот вектор проходит через несколько слоев, постепенно увеличивая размерность до 784 (28×28 пикселей). Функция активации на выходе — Tanh, которая нормализует значения пикселей в диапазон от -1 до 1.

```python
class Generator(nn.Module):
    def __init__(self, latent_dim=128):
        super(Generator, self).__init__()
        self.model = nn.Sequential(
            nn.Linear(latent_dim, 256),
            nn.BatchNorm1d(256),
            nn.ReLU(),
            nn.Linear(256, 512),
            nn.BatchNorm1d(512),
            nn.ReLU(),
            nn.Linear(512, 1024),
            nn.BatchNorm1d(1024),
            nn.ReLU(),
            nn.Linear(1024, 28*28),
            nn.Tanh()
        )
    
    def forward(self, z):
        return self.model(z).view(-1, 1, 28, 28)
```

**Дискриминатор (Discriminator)**
Дискриминатор получает на вход изображение размером 1×28×28 и определяет, является ли оно реальным или поддельным. Архитектура представляет собой последовательность полносвязных слоев с функциями активации LeakyReLU, которая помогает избежать проблемы "исчезающих градиентов". На выходе используется сигмоидная функция активации, выдающая вероятность принадлежности к классу "реальных" изображений (от 0 до 1).

```python
class Discriminator(nn.Module):
    def __init__(self):
        super(Discriminator, self).__init__()
        self.model = nn.Sequential(
            nn.Linear(28*28, 512),
            nn.LeakyReLU(0.2),
            nn.Linear(512, 256),
            nn.LeakyReLU(0.2),
            nn.Linear(256, 1),
            nn.Sigmoid()
        )
    
    def forward(self, img):
        img_flat = img.view(img.size(0), -1)
        return self.model(img_flat)
```

#### **3.2 Процесс обучения**

Обучение GAN включает в себя одновременную оптимизацию двух сетей с использованием состязательной игры.

```python
def train_gan(generator, discriminator, dataloader, epochs, latent_dim, device):
    # Функции потерь и оптимизаторы
    adversarial_loss = nn.BCELoss()
    optimizer_G = optim.Adam(generator.parameters(), lr=0.0002, betas=(0.5, 0.999))
    optimizer_D = optim.Adam(discriminator.parameters(), lr=0.0002, betas=(0.5, 0.999))

    for epoch in range(epochs):
        for i, (imgs, _) in enumerate(dataloader):
            # Метки для реальных и фейковых данных
            real = torch.ones(imgs.size(0), 1).to(device)
            fake = torch.zeros(imgs.size(0), 1).to(device)

            # --- Обучение дискриминатора ---
            real_imgs = imgs.to(device)
            # Реальная потеря
            real_loss = adversarial_loss(discriminator(real_imgs), real)
            # Фейковая потеря
            z = torch.randn(imgs.size(0), latent_dim).to(device)
            fake_imgs = generator(z)
            fake_loss = adversarial_loss(discriminator(fake_imgs.detach()), fake)
            # Полная потеря дискриминатора
            d_loss = (real_loss + fake_loss) / 2

            optimizer_D.zero_grad()
            d_loss.backward()
            optimizer_D.step()

            # --- Обучение генератора ---
            z = torch.randn(imgs.size(0), latent_dim).to(device)
            fake_imgs = generator(z)
            # Генератор старается обмануть дискриминатор
            g_loss = adversarial_loss(discriminator(fake_imgs), real)

            optimizer_G.zero_grad()
            g_loss.backward()
            optimizer_G.step()

        # Визуализация прогресса каждые 10 эпох
        if epoch % 10 == 0:
            print(f"[Эпоха {epoch}/{epochs}] [D loss: {d_loss.item():.4f}] [G loss: {g_loss.item():.4f}]")
            visualize_generated_images(generator, epoch, latent_dim, device)
```

#### **3.3 Результаты генерации**

В процессе обучения наблюдалось улучшение качества генерируемых изображений. На ранних эпохах генератор создавал случайный шум, не имеющий структуры. К середине обучения (эпохи 20-30) начали проявляться смутные очертания цифр и базовые формы. К 50-й эпохе генератор научился создавать узнаваемые цифры с четкими контурами. К финальным эпохам обучения сгенерированные изображения рукописных цифр стали визуально неотличимы от реальных образцов из датасета MNIST.

### **4. Практическое применение GAN-моделей**

#### **4.1 Компьютерное зрение и генерация изображений**
GAN широко применяются для задач компьютерного зрения:
* **Создание фотореалистичных изображений**: от портретов несуществующих людей до пейзажей и объектов.
* **Увеличение разрешения (Super-Resolution)**: модель SRGAN (Super-Resolution GAN) специализируется на повышении разрешения изображений с восстановлением деталей.
* **Перенос стиля**: методы, такие как CycleGAN, позволяют преобразовывать изображения из одного домена в другой (например, превращать фотографии лошадей в зебр или менять времена года на пейзажах).
* **Аугментация данных**: GAN используются для создания синтетических данных для обучения моделей в различных областях, включая медицину и задачи с несбалансированными выборками. Например, исследования показывают, что применение GAN для синтеза данных при преодолении дисбаланса классов позволяет улучшить полноту (recall) моделей в среднем на 60% по сравнению с классическими методами вроде SMOTE.

#### **4.2 Медицина и биология**
GAN находят широкое применение в медицинской визуализации:
* **Синтез медицинских изображений**: GAN используются для генерации синтетических изображений МРТ (например, для визуализации опухолей головного мозга без использования контрастного вещества) и КТ.
* **Улучшение качества**: модели помогают повышать качество МРТ-снимков, снижать уровень шума и улучшать структурную четкость.
* **Создание синтетических данных для обучения**: GAN могут генерировать синтетические медицинские изображения для обучения диагностических моделей, что особенно ценно в условиях дефицита реальных данных или для сохранения конфиденциальности пациентов.

#### **4.3 Безопасность и обнаружение подделок**
GAN играют двойную роль в области безопасности:
* **Создание DeepFakes**: GAN используются для подмены лиц и создания реалистичных видеоподделок.
* **Обнаружение подделок**: В то же время GAN могут применяться для распознавания синтезированных изображений и борьбы с дипфейками.

#### **4.4 Обнаружение аномалий**
GAN могут быть использованы для обнаружения аномалий в данных. Модель обучается на "нормальных" данных, а затем аномалии выявляются как точки, которые генератор не может хорошо воспроизвести.

#### **4.5 Генерация 3D-объектов**
Существуют модели GAN, способные генерировать трёхмерные объекты на основе 2D-изображений.
