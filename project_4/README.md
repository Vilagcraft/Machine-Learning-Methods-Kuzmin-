## Инструкция по запуску программы (Windows)

### Требования
- **Операционная система**: Windows 10/11
- **Python**: версия 3.8 – 3.11 (рекомендуется 3.10)
- **Доступ в интернет** (для автоматической загрузки датасета CIFAR-10)

### Установка Python и зависимостей

1. **Скачайте Python** с официального сайта: https://www.python.org/downloads/  
   При установке **обязательно отметьте** галочку `Add Python to PATH`.

2. **Проверьте установку** в командной строке (Win+R → `cmd`):
   ```cmd
   python --version
   pip --version
   ```

3. **Установите необходимые библиотеки** одной командой:
   ```cmd
   pip install tensorflow numpy matplotlib pillow scikit-learn
   ```
   > Если скорость низкая, добавьте флаг `-i https://pypi.tuna.tsinghua.edu.cn/simple`

### Запуск программы

1. Скачайте файл `cnn_classifier.py` (или создайте его и скопируйте код из предыдущего сообщения).

2. Откройте командную строку в папке с файлом:
   ```cmd
   cd C:\путь\к\папке
   ```

3. Запустите обучение:
   ```cmd
   python cnn_classifier.py
   ```

4. Что произойдёт:
   - При первом запуске автоматически загрузятся данные CIFAR-10 (~160 МБ) и создастся папка `cifar10_4classes` с 4 классами (самолёт, автомобиль, птица, кошка).
   - Обучится свёрточная сеть (CNN), затем полносвязная (MLP) – для сравнения.
   - Появятся графики точности и потерь.
   - Программа покажет пример распознавания одного изображения.

5. **Результаты** сохраняются:
   - `best_cnn.h5` – лучшая CNN модель
   - `cnn_final.h5` – финальная CNN модель
   - `mlp_final.h5` – модель MLP

### Использование своего датасета

1. Создайте структуру папок:
   ```
   my_dataset/
   ├── train/
   │   ├── class_1/
   │   ├── class_2/
   │   ├── class_3/
   │   └── class_4/
   └── val/                (такие же подпапки)
   ```
2. В коде измените переменные:
   ```python
   DATA_DIR = './my_dataset'
   CLASS_NAMES = ['class_1', 'class_2', 'class_3', 'class_4']
   ```
3. Закомментируйте вызов `prepare_data()`.
4. Запустите повторно.

## Ответы на вопросы

### 1. Математическая операция свёртки. Свёртка матриц.

Свёртка двух матриц – это поэлементное умножение ядра (небольшой матрицы) на перекрывающуюся область входной матрицы с последующим суммированием результатов. Формула для дискретной 2D-свёртки:

(f * g)(i,j) = \sum_{m} \sum_{n} f(i+m, j+n) \cdot g(m,n)

где ( f ) – входное изображение, ( g ) – ядро. Ядро скользит по входу, вычисляя скалярное произведение на каждом шаге. В CNN это используется для выделения признаков: например, ядро 3×3 с определёнными весами может обнаруживать вертикальные края.

### 2. Взаимодействие ядер свёртки и карт признаков

Каждое ядро свёртки создаёт **одну карту признаков** (feature map). Если на первом слое используется 32 ядра – получаем 32 карты признаков. Каждая карта показывает, где на изображении присутствует определённый паттерн (например, горизонтальная линия, угол, цветовое пятно). При переходе к следующим слоям ядра взаимодействуют уже не с исходными пикселями, а с предыдущими картами признаков – так сеть строит иерархию абстракций: от простых линий к формам и даже целым объектам.

### 3. Операция подвыборки (subsample)

Подвыборка (пулинг, pooling) уменьшает пространственный размер карт признаков. Чаще всего используют **MaxPooling** – окно (например 2×2) скользит по карте и оставляет только максимальное значение в каждом блоке. Это даёт:
- Снижение вычислительной нагрузки.
- Борьбу с переобучением.
- Инвариантность к небольшим сдвигам и искажениям (важно, когда объект на изображении смещён).

### 4. Распространённые функции активации

- **ReLU** (Rectified Linear Unit): \( f(x) = \max(0, x) \) – используется чаще всего внутри свёрточных слоёв. Очень быстро вычисляется, устраняет проблему затухающих градиентов.
- **Sigmoid**: \( \sigma(x) = 1/(1+e^{-x}) \) – выход в диапазоне (0,1), но насыщается и приводит к исчезновению градиента.
- **Tanh**: \( \tanh(x) \) – от -1 до 1, аналогичные недостатки.
- **Softmax**: используется в последнем слое для многоклассовой классификации – превращает выходы в вероятности классов.

### 5. Общий принцип работы свёрточных моделей

1. **Свёрточные слои** извлекают пространственные признаки (края, текстуры, части объектов).
2. **Пулинг** уменьшает размерность и добавляет робастность.
3. **Полносвязные слои** в конце выполняют классификацию на основе высокоуровневых признаков.
4. Веса ядер обучаются методом обратного распространения ошибки (градиентный спуск).

В отличие от MLP, свёрточная сеть использует **разделяемые веса** (одно и то же ядро применяется ко всей картинке), что резко сокращает число параметров и позволяет сети обобщать.

### 6. Типичная архитектура CNN и её функционирование

**Пример архитектуры VGG-подобной сети** (используется в коде):

```
Вход (64×64×3)
→ Conv2D(32, 3×3, ReLU) → MaxPool(2×2)   → размер 32×32×32
→ Conv2D(64, 3×3, ReLU) → MaxPool(2×2)   → размер 16×16×64
→ Conv2D(128,3×3, ReLU) → MaxPool(2×2)  → размер 8×8×128
→ Flatten → Dense(128, ReLU) → Dropout → Dense(4, Softmax)
```

Функционирование: первые слои выделяют примитивные признаки, средние – сочетания (например, линии, образующие круг), последние свёрточные слои – целые части объекта (колесо, крыло). Полносвязный слой принимает решение.

### 7. Типичная архитектура CNN и её обучение

**Обучение CNN** включает:
- Прямой проход (forward pass): вычисление выходов всех слоёв.
- Вычисление функции потерь (например, категориальная кросс-энтропия).
- Обратный проход (backpropagation): расчёт градиентов с помощью правила цепочки.
- Обновление весов ядер и полносвязных слоёв (оптимизатор Adam/SGD).
- Повторение для мини-пакетов изображений в течение нескольких эпох.

Важно использовать **регуляризацию** (Dropout, аугментация данных, ранняя остановка) для борьбы с переобучением.

### 8. Преимущества свёрточных сетей

- **Пространственная инвариантность** – сеть распознаёт объект независимо от его положения (благодаря свёртке и пулингу).
- **Разделяемые веса** – на порядок меньше параметров, чем у полносвязной сети того же размера.
- **Иерархическое извлечение признаков** – автоматическое построение сложных абстракций из простых.
- **Высокая точность** в задачах компьютерного зрения (ImageNet, распознавание лиц, медицинская диагностика).

### 9. Недостатки свёрточных сетей

- **Требовательность к данным** – для обучения с нуля нужны десятки тысяч размеченных изображений.
- **Вычислительная сложность** – обучать CNN дольше, чем MLP, особенно на больших изображениях.
- **Чувствительность к масштабу и повороту** – обычная CNN не инвариантна к повороту или масштабированию (требуется аугментация).
- **Сложность интерпретации** – трудно понять, почему сеть приняла то или иное решение.

### 10. Макропараметры свёрточных сетей и проблема их подбора

**Гиперпараметры** (макропараметры):
- Количество и размер ядер (например, 32 ядра 3×3).
- Количество свёрточных блоков.
- Размер пулинга (2×2 или 3×3).
- Функция активации (обычно ReLU).
- Скорость обучения, количество эпох, размер батча.
- Дропаут (вероятность отключения нейронов).

**Проблема подбора**: гиперпараметры сильно влияют на результат, но оптимальные значения зависят от данных. Используют:
- **Grid search / Random search** – перебор.
- **Байесовская оптимизация** – более умный подбор.
- **Эмпирические правила** (например, удваивать количество ядер после каждого пулинга).

### 11. Потенциал распараллеливания процессов обучения и функционирования CNN

- **Прямой проход** – свёртки легко параллелится, так как операции над разными ядрами или разными участками изображения независимы (используются GPU с тысячами ядер).
- **Обучение** – может быть синхронно распараллелено по нескольким GPU (data parallelism: распределение мини-батчей) или по узлам (distributed training).
- **Функционирование** – одна картинка проходит быстро даже на CPU благодаря оптимизированным библиотекам. Для большого потока применяются батчи.

**Итог**: CNN идеально подходят для GPU-ускорения, что делает их практичными даже для больших моделей.

### 12. Практическое применение свёрточных моделей

- **Классификация изображений** (ваше задание) – определение объекта на фото.
- **Детекция объектов** (YOLO, Faster R-CNN) – поиск и локализация всех объектов на сцене.
- **Сегментация** (U-Net, Mask R-CNN) – выделение контуров каждого объекта.
- **Медицина** – анализ рентгеновских снимков, МРТ, гистологии.
- **Автономные автомобили** – распознавание дорожных знаков, пешеходов.
- **Безопасность** – распознавание лиц, анализ видео.
- **Генерация изображений** (GAN, генеративно-состязательные сети).
