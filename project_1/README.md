# 🧠 Классификатор изображений на базе многослойного перцептрона

Проект по курсу «Методы машинного обучения». Реализовано приложение для обучения и использования нейросети (feed‑forward) для классификации цветных изображений на **4 класса**: `birds`, `cats`, `dogs`, `fish`.

## 📦 Требования

```markdown
- Python 3.8+
- TensorFlow / Keras
- NumPy, Matplotlib, OpenCV

Установка зависимостей:
```bash
pip install -r requirements.txt
```

## 📁 Структура данных

Обучающая выборка организована в виде каталогов:

```
data/
├── birds/
│   ├── img1.jpg
│   └── ...
├── cats/
├── dogs/
└── fish/
```

> **Важно:** количество изображений в классах должно быть примерно одинаковым, иначе модель будет смещена в сторону доминирующего класса (данную проблему решил благодаря sklearn, но лучше не рисковать).

## 🚀 Обучение модели

Запустите обучение из корневой папки проекта:

```bash
python app.py train --data_dir ./data --epochs 20 --model_path model.h5
```

**Параметры:**
- `--data_dir` – путь к корневому каталогу с подкаталогами классов.
- `--epochs` – количество эпох (по умолчанию 20).
- `--model_path` – имя файла для сохранения модели (по умолчанию `model.h5`).

### Что происходит во время обучения?

- Данные автоматически делятся на **обучающую (80%)** и **валидационную (20%)** выборки.
- Применяется **аугментация** (повороты, сдвиги, масштабирование) для улучшения обобщения.
- Лучшая модель (по точности на валидации) сохраняется в указанный файл.
- После обучения строятся графики потерь и точности, выводится итоговая точность на валидации.
- Создаётся файл `class_names.txt` со списком классов в порядке, соответствующем предсказаниям модели.

## 🔍 Предсказание для одного изображения

**Результат:**

## 🧠 Архитектура модели

Используется **сверточная нейронная сеть (CNN)** – полносвязная нейронная сеть:

- Входной слой: `Flatten` – преобразование изображения 128×128×3 в вектор размером 49152.
- Скрытые слои: `Dense(128, relu)` + `Dropout(0.5)`, затем `Dense(128, relu)` + `Dropout(0.5)`.
- Выходной слой: `Dense(4, softmax)`.

Модель компилируется с оптимизатором Adam и функцией потерь `categorical_crossentropy`.

> ⚠️ **Примечание:** MLP не обладает пространственной инвариантностью и требует много данных. Поэтому использовал свёрточную сеть (CNN).

## 📌 Важные замечания

- **Классы:** модель распознаёт только `birds`, `cats`, `dogs`, `fish`. Для других объектов результат будет случайным с низкой уверенностью.
- **Надёжность предсказания:** если максимальная вероятность ниже **0.6–0.7**, результат следует считать ненадёжным.
- **Файлы:** убедитесь, что `model.h5` и `class_names.txt` находятся в папке проекта (они создаются при обучении).
- **Формат модели:** если вы сохранили модель с расширением `.keras`, укажите `--model_path model.keras` соответственно.
- **Баланс классов:** при сильном дисбалансе (например, 60 birds против 10000 собак) модель будет предсказывать только доминирующий класс. Рекомендуется использовать подвыборку (`undersampling`) или взвешивание классов.

# Ответы на вопросы по нейронным сетям и машинному обучению

## 1) Модель формального нейрона

Формальный нейрон — это математическая абстракция биологического нейрона. Он принимает на вход несколько сигналов \(x_1, x_2, ..., x_n\), каждому из которых соответствует вес \(w_i\). Взвешенная сумма входов суммируется с порогом (смещением) \(b\), затем результат пропускается через функцию активации \(f\). Выход нейрона:  
\[
y = f\left(\sum_{i=1}^{n} w_i x_i + b\right)
\]
Модель была предложена Маккалоком и Питтсом (1943).

## 2) Функции активации. Распространённые примеры

Функция активации вносит нелинейность, позволяя нейронной сети аппроксимировать сложные зависимости. Распространённые примеры:

- **Сигмоида**: \(f(x) = \frac{1}{1+e^{-x}}\). Выход в диапазоне (0,1). Используется в выходных слоях для бинарной классификации.
- **Tanh (гиперболический тангенс)**: \(f(x) = \frac{e^{x}-e^{-x}}{e^{x}+e^{-x}}\). Выход в (-1,1). Нулевое центрирование.
- **ReLU (Rectified Linear Unit)**: \(f(x) = \max(0, x)\). Проста в вычислении, не страдает от насыщения градиента.
- **Leaky ReLU**: \(f(x) = \alpha x\) для \(x<0\) (например, \(\alpha=0.01\)). Решает проблему «умирающих» нейронов ReLU.
- **Softmax**: используется в многоклассовой классификации; превращает выходной вектор в распределение вероятностей.

## 3) Модель многослойного персептрона

Многослойный персептрон (MLP) — это feed-forward нейронная сеть, состоящая из одного входного слоя, одного или нескольких скрытых слоёв и выходного слоя. Каждый нейрон слоя соединён со всеми нейронами следующего слоя (полносвязная архитектура). Благодаря нелинейным функциям активации MLP способен аппроксимировать любую непрерывную функцию (универсальная теорема аппроксимации). Обучение происходит по алгоритму обратного распространения ошибки.

## 4) Задача классификации

Классификация — это задача машинного обучения, в которой объекту (набору признаков) необходимо присвоить одну из дискретных меток (классов). Примеры: распознавание рукописных цифр, определение вида животного на фото, фильтрация спама. В зависимости от количества классов различают бинарную (2 класса) и многоклассовую (более 2) классификацию. Модель классификатора выдаёт предсказанный класс или вероятности принадлежности к каждому классу.

## 5) Пространство признаков и пространство состояний

- **Пространство признаков (feature space)** — многомерное пространство, в котором каждый объект представлен вектором признаков. Размерность пространства равна числу признаков. Например, для изображения 128×128 пикселей пространство признаков имеет 16384 измерения (если использовать яркости пикселей).

- **Пространство состояний (state space)** — более широкое понятие, обычно используется в динамических системах и обучении с подкреплением. Это множество всех возможных состояний системы. В контексте нейросетей пространство состояний может относиться к скрытым представлениям (активациям) слоёв сети.

## 6) Функция потерь. Распространённые примеры

Функция потерь (loss function) количественно оценивает разницу между предсказаниями модели и истинными значениями. Цель обучения — минимизировать значение функции потерь.

- **Среднеквадратичная ошибка (MSE)**: \(L = \frac{1}{N}\sum_{i=1}^{N}(y_i - \hat{y}_i)^2\). Используется в регрессии.
- **Бинарная кросс-энтропия**: \(L = -\frac{1}{N}\sum_{i=1}^{N}[y_i \log(\hat{y}_i) + (1-y_i)\log(1-\hat{y}_i)]\). Для бинарной классификации.
- **Категориальная кросс-энтропия (Categorical Cross-Entropy)**: \(L = -\frac{1}{N}\sum_{i=1}^{N}\sum_{c=1}^{C} y_{i,c} \log(\hat{y}_{i,c})\). Для многоклассовой классификации.
- **Hinge loss**: используется в SVM.

## 7) Обучение «с учителем»

Обучение с учителем (supervised learning) — это подход, при котором модель обучается на размеченном наборе данных, где каждому входному объекту \(x\) соответствует правильный ответ \(y\) (метка). Модель находит отображение \(f: X \to Y\), минимизируя ошибку на обучающей выборке. Типичные задачи: классификация и регрессия. Примеры: распознавание цифр по картинкам (метки — цифры), предсказание цены дома (метка — цена).

## 8) Метод обратного распространения ошибки

Обратное распространение ошибки (backpropagation) — это алгоритм для вычисления градиента функции потерь по весам нейронной сети. Состоит из двух проходов:

1. **Прямой проход (forward pass)**: входные данные проходят через сеть, вычисляются выходы и значение функции потерь.
2. **Обратный проход (backward pass)**: начиная с выходного слоя, с помощью правила цепочки вычисляются частные производные по каждому весу. Полученные градиенты используются для обновления весов (например, методом градиентного спуска).

Этот алгоритм лежит в основе обучения всех современных нейросетей.

## 9) Проблемы градиентных методов обучения. Overfitting, underfitting

- **Overfitting (переобучение)**: модель слишком точно запоминает обучающую выборку, включая шумы, и плохо обобщается на новые данные. Признаки: низкая ошибка на обучении, но высокая на валидации. Методы борьбы: регуляризация, dropout, аугментация данных, ранняя остановка.

- **Underfitting (недообучение)**: модель слишком проста и не способна уловить закономерности даже на обучающей выборке. Признаки: высокая ошибка и на обучении, и на валидации. Решения: усложнение модели (добавление слоёв, нейронов), уменьшение регуляризации, увеличение числа эпох.

## 10) Проблемы градиентных методов обучения. Локальные минимумы

Градиентные методы могут застревать в локальных минимумах (точках, где градиент равен нулю, но значение функции выше глобального минимума). Для глубоких сетей проблема локальных минимумов не столь остра, как проблема седловых точек (где градиент также нулевой, но есть направления как увеличения, так и уменьшения функции). Методы преодоления: использование стохастического градиентного спуска с моментом (momentum), адаптивные оптимизаторы (Adam, RMSprop), случайная инициализация весов, периодическое увеличение скорости обучения.

## 11) Проблемы градиентных методов обучения. Выбор шага обучения

Шаг обучения (learning rate) определяет величину обновления весов на каждой итерации:

- **Слишком большой шаг**: градиентный спорт «перепрыгивает» через минимум, функция потерь расходится или колеблется, не сходясь.
- **Слишком маленький шаг**: обучение идёт очень медленно, возможно застревание в локальном минимуме или седловой точке.

Решения: использовать адаптивные методы (Adam, AdaGrad), уменьшать скорость обучения по расписанию (learning rate decay), применять циклические скорости обучения.

## 12) Нормализация данных

Нормализация (масштабирование) данных — это приведение признаков к единому диапазону (например, [0,1] или стандартизация к нулевому среднему и единичной дисперсии). Цели:

- Ускорение сходимости градиентных методов (поскольку все признаки вносят вклад одного порядка).
- Предотвращение доминирования признаков с большими числовыми значениями.
- Улучшение обобщающей способности модели.

Распространённые методы: MinMaxScaler (x' = (x - min)/(max - min)), StandardScaler (x' = (x - μ)/σ).

## 13) Регуляризация

Регуляризация — это набор техник, направленных на уменьшение переобучения путём добавления штрафа за сложность модели.

- **L1-регуляризация (Lasso)**: добавляет к функции потерь \(\lambda \sum |w_i|\). Способствует разреженности весов (обнуляет неважные).
- **L2-регуляризация (Ridge, weight decay)**: добавляет \(\lambda \sum w_i^2\). Уменьшает абсолютные значения весов, но не обнуляет.
- **Dropout**: во время обучения случайно «выключает» часть нейронов, делая сеть устойчивее.
- **Early stopping**: остановка обучения, когда ошибка на валидации перестаёт снижаться.
- **Аугментация данных**: искусственное увеличение выборки путём преобразований (повороты, сдвиги).

## 14) Чувствительность и специфичность классификатора

Эти метрики используются для оценки качества бинарного классификатора (классы: положительный P и отрицательный N).

- **Чувствительность (Sensitivity, Recall, True Positive Rate)**: доля правильно предсказанных положительных объектов среди всех реально положительных.  
  \(\text{Sensitivity} = \frac{TP}{TP+FN}\). Показывает, насколько хорошо модель находит положительный класс.

- **Специфичность (Specificity, True Negative Rate)**: доля правильно предсказанных отрицательных объектов среди всех реально отрицательных.  
  \(\text{Specificity} = \frac{TN}{TN+FP}\). Показывает, насколько хорошо модель избегает ложных срабатываний.

В многоклассовой классификации часто вычисляют микро- или макроусреднение этих метрик.

## 15) Практическое применение feed-forward моделей

Полносвязные сети (feed-forward, MLP) применяются в задачах, где нет явной пространственной или временной структуры, либо размерность входных данных невелика:

- **Классификация табличных данных** (например, кредитный скоринг, диагностика заболеваний по анализам).
- **Распознавание образов** на небольших изображениях (например, рукописные цифры MNIST).
- **Аппроксимация функций** (регрессия).
- **Вспомогательные компоненты** в более сложных архитектурах (например, головы классификации в CNN или встраивающие слои в трансформерах).

Для работы с изображениями, последовательностями или графами обычно используют более специализированные архитектуры (CNN, RNN, GNN), но MLP остаётся базовым и полезным инструментом.
